# Data Sources

## Yahoo Finance (yfinance)
- **What**: Stock prices, crypto prices, commodity futures, VIX, DXY
- **API Key**: None required
- **Rate Limits**: ~2000 requests/hour. Batch downloads mitigate this.
- **Quirks**:
  - S&P tickers with dots use dashes in yfinance (BRK.B -> BRK-B)
  - Some commodity futures may have gaps on holidays
  - Crypto trades 24/7, stocks only market hours
- **Package**: `yfinance`

## FRED (Federal Reserve Economic Data)
- **What**: Fed Funds Rate, M2, CPI, Treasury yields, credit spreads
- **API Key**: Required (free). Get at https://fred.stlouisfed.org/docs/api/api_key.html
- **Rate Limits**: 120 requests/minute (generous)
- **Quirks**:
  - Many series update monthly, not daily
  - Real-time data may have 1-2 day lag
  - Series IDs: FEDFUNDS, M2SL, CPIAUCSL, DGS2, DGS10, BAMLH0A0HYM2
- **Package**: `fredapi`

## CoinGecko
- **What**: Crypto metadata, market cap rankings (used for universe selection)
- **API Key**: None for free tier
- **Rate Limits**: 10-30 calls/minute (free tier)
- **Quirks**:
  - Free tier has aggressive rate limiting
  - We use yfinance for actual crypto prices (more reliable)
  - CoinGecko IDs differ from ticker symbols
- **Package**: `pycoingecko`

## Wikipedia
- **What**: S&P 500 and S&P 400 constituent lists
- **API Key**: None
- **Rate Limits**: None practical
- **Quirks**:
  - Table format may change without notice
  - Column names vary between S&P 500 and S&P 400 pages
  - Updated in near-real-time when index changes happen
- **Package**: `pandas.read_html`

## Data Refresh Schedule
| Data Type | Frequency | Best Time |
|-----------|-----------|-----------|
| Stock prices | Daily | After 4:30 PM ET |
| Crypto prices | Daily | Anytime (24/7 market) |
| Commodity prices | Daily | After 5:00 PM ET |
| FRED macro | Daily check | Morning (updates vary) |
| Stock fundamentals | Weekly | Weekend |
| Stock universe | Weekly | Weekend |

## Storage
All data stored in SQLite at `.tmp/druckenmiller.db`. This file is disposable - can be fully regenerated by running `python -m tools.daily_pipeline`.
